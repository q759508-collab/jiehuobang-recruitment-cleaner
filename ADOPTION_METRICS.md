# 使用情况观察指标

第一版开源后，先观察有没有真实需求，再决定是否做更大的系统。

## 公开平台指标

每周看一次：

- GitHub Star。
- GitHub Fork。
- GitHub Issue。
- Gitee Star / 收藏。
- Gitee Fork。
- Gitee Issue。
- 是否有人提交真实样本。
- 是否有人问批量、Excel、Word、网页或微信采集。

## 有效需求信号

强信号：

- 用户愿意提供真实招聘样本。
- 用户指出具体识别错误。
- 用户要求批量处理。
- 用户要求 Word 或 Excel 导出。
- 县城招聘站、劳务公司、人力资源从业者主动询问怎么用。

弱信号：

- 只点赞或收藏。
- 只说有意思，但不提供样本。
- 只问能不能采微信，但不试清洗。

## 下一步判断

如果 5 个试用者里有 3 个需要批量或 Excel：

```text
优先做批量清洗脚本和 Excel 导出。
```

如果 5 个试用者里有 3 个需要网页：

```text
做一个轻量网页演示，不先做完整平台。
```

如果 5 个试用者里有 3 个需要微信采集：

```text
先研究桌面端本地采集，不做群发，不做营销自动化。
```

如果 2 周没有真实样本反馈：

```text
暂停扩大开发，只保留开源项目。
```
