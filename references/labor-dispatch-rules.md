# 劳务发单文本专项规则

## 适用场景

微信群里大量招聘文本不是普通企业直招，而是劳务、派遣、供应商发单。

这些文本常见字段：

- 员工薪资。
- 供应商政策。
- 返费。
- 开票扣税。
- 商保。
- 体检费。
- 压薪。
- 卡点。
- 自离无费用。
- 名单报备。
- 政策咨询。
- 财务结款。
- 投诉电话。

## 结构化建议

遇到这类信息，`data` 中可以增加：

```json
{
  "worker_pay": "",
  "supplier_policy": "",
  "rebate_policy": "",
  "invoice_tax_policy": "",
  "insurance_fee": "",
  "medical_exam_fee": "",
  "wage_hold_policy": "",
  "attendance_policy": "",
  "contacts": []
}
```

联系人建议结构：

```json
{
  "role": "名单报备",
  "name": "张老师",
  "phone": "13800000000",
  "wechat_same_as_phone": true
}
```

## 联系人角色

识别这些角色：

- 名单报备。
- 政策咨询。
- 异常反馈。
- 财务对接。
- 财务结款。
- 业务对接。
- 合作热线。
- 投诉电话。
- 现场负责。
- 面试老师。
- 带队老师。

## 风险和备注

### 高优先级风险

写入 `risk_flags`：

- 体检费不报销。
- 押工资。
- 不满 7 天无工资。
- 旷离、自离无工资。
- 急辞扣款。
- 需先交费。
- 身份证照片、流水、社保、公积金证明等敏感要求。

### 业务备注

写入 `warnings` 或 `notes`：

- 返费政策。
- 开票扣税。
- 商保费用。
- 水电费平摊。
- 住宿费。
- 需要提前申请离职。
- 工价有效期。
- 暑假工可接收。
- 学生证明或非学生证明要求。

## 数字异常规则

### 年龄粘连

`2052岁` 这类写法不要识别成 2052 岁。

如果上下文是年龄要求，推测为：

```text
20-52岁
```

但必须写入 `warnings`：

```text
原文年龄疑似粘连：2052岁，可能为20-52岁，需人工确认。
```

### 薪资范围粘连

`200220元` 可能表示 `200-220元`。

`58006500元` 可能表示 `5800-6500元`。

不要直接当成单个金额。

必须写入 `warnings`：

```text
原文薪资疑似范围粘连，需人工确认。
```

## 日期和时间

- `6.8`、`6月8日`、`6/8` 识别为日期。
- `下午13点半` 识别为 `13:30`，但提示表达重复。
- `8.30-20.30` 识别为 `08:30-20:30`。
- `20.30-8-30` 识别为 `20:30-08:30`，并提示原文格式异常。

## 岗位类型

劳务发单中常见岗位：

- 普工。
- 操作工。
- 包装工。
- 质检员。
- 绕线普工。
- 拣货。
- 打磨工。
- 辅助工。
- 技术员。

## 简版多企业政策

遇到这种格式：

```text
单#湖北联想
年龄:18-45男女不限
供应商政策: 打卡大于101工时2300+
员工工价:20
```

应输出 `job_batch`。

每个 `单#企业名` 是一个 `items` 项。

这种简版通常缺：

- 具体岗位。
- 工作时间。
- 工作地址。
- 吃住情况。
- 联系人角色细节。

必须把这些写入 `missing_fields`。

公共政策如含税、卡点、投诉不结费、水军处理，应写到顶层 `warnings` 或 `risk_flags`。

## 输出注意

清洗这类文本时不要只输出岗位字段，还要保留：

- 发单政策。
- 供应商利润。
- 结算方式。
- 风险条款。
- 多联系人。

## 批量发单混入高风险信息

如果一条批量发单里混入疑似非普通招聘内容，例如高额补偿志愿者、代理招募、身体条件筛选等：

- 不要合并进普通岗位列表。
- 单独放入 `risk_flags` 或 `warnings`。
- 建议人工审核。
- 如输出 `job_batch`，可以在顶层写明 `mixed_risk_content: true`。
