# 真实样本采集与脱敏指南

## 目标

收集 20-50 条真实招聘群文本，用来验证 Skill 是否真的能处理业务场景。

## 可以收集什么

- 岗位招聘。
- 求职者自荐。
- 招人需求。
- 临时活。
- 一人公司接单。
- 无关或低价值群聊信息。

## 不要直接保存什么

不要直接保存真实敏感信息：

- 真实手机号。
- 真实微信号。
- 真实姓名。
- 身份证。
- 详细住址。
- 企业内部敏感信息。

## 脱敏规则

### 手机号

把真实手机号替换成：

```text
13800000000
13800000000
```

不要只打星号，因为测试联系方式识别时需要保留格式。

### 微信号

把真实微信号替换成：

```text
wechat001
hr2026
job_test_01
```

### 姓名

把真实姓名替换成：

```text
张姐
李老师
王总
赵经理
```

### 详细地址

可以保留城市和区域，隐藏门牌号。

例：

```text
成都新都某食品厂
武汉光谷某园区
```

## 每条样本建议保留的信息

- 原文。
- 来源类型：微信群、朋友圈、私聊、表格。
- 你认为它是什么类型。
- 你希望提取哪些字段。
- 当前人工整理时最麻烦的地方。

## 采集模板

```text
【来源】微信群 / 朋友圈 / 私聊 / 表格
【类型】岗位 / 求职者 / 招人需求 / 临时活 / 无关
【原文】
粘贴脱敏后的原文

【人工期望】
你希望 AI 整理出什么

【备注】
哪里容易识别错
```

## 第一批样本目标

建议第一批收集：

- 8 条岗位招聘。
- 5 条求职者自荐。
- 4 条招人需求。
- 3 条临时活。
- 3 条风险信息。
- 3 条无关信息。

合计约 26 条。
