# 反馈入口设计

## 第一阶段反馈入口

建议第一阶段使用两个入口：

1. 微信私聊或微信群。
2. GitHub / Gitee Issue。

原因：

- 微信适合熟人试用。
- Issue 适合开源用户沉淀问题。

## 微信反馈格式

让试用者直接按这个格式发：

```text
【原文】
脱敏后的招聘信息

【哪里不对】
类型 / 字段 / 风险 / 文案

【我希望】
希望整理成什么样

【是否需要批量或 Excel】
需要 / 不需要
```

## Issue 反馈类型

已准备两个模板：

- `.github/ISSUE_TEMPLATE/cleaning-error.yml`
- `.github/ISSUE_TEMPLATE/feature-request.yml`

## 反馈优先级

### P0

- 年龄、日期、时间识别错误。
- 押金、服装费、报名费风险漏识别。
- 类型判断完全错误。

### P1

- 城市、薪资、联系方式漏提取。
- 发布文案不自然。
- 表格字段不够用。

### P2

- 想要批量。
- 想要 Excel。
- 想要网页。
- 想要微信采集。

## 反馈判断

如果 5 个试用者里有 3 个都问批量或 Excel，下一步优先做批量 CSV 工具。

如果 5 个试用者里有 3 个都问网页，下一步做轻量网页演示。

如果没人愿意给真实样本，暂停继续开发。
