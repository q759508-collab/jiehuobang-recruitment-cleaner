# AI 智能体发现与使用设计

## 目标

这个 Skill 不只是给人下载，也要让 AI 智能体更容易搜索、理解和使用。

## 发布平台

优先：

- GitHub。
- Gitee。

原因：

- GitHub 更利于 AI 搜索和代码索引。
- Gitee 对国内用户访问更友好。

## 仓库命名

推荐仓库名：

```text
jiehuobang-recruitment-cleaner
```

中文名：

```text
接活帮招聘信息清洗 Skill
```

## 关键词

README 和仓库描述里要出现：

- Codex Skill
- 招聘信息清洗
- 微信群招聘
- 私域招聘
- 劳务招聘
- 求职信息整理
- 招人需求
- 招聘 JSON
- 招聘表格
- 招聘风险识别
- recruitment cleaner
- Chinese recruitment
- WeChat group recruiting

## AI 友好文件

为了让 AI 智能体容易理解，仓库里要保留：

- `README.md`：人和 AI 都先读。
- `SKILL.md`：Codex Skill 主入口。
- `QUICKSTART.md`：最快使用方式。
- `demo-cases.md`：典型输入输出。
- `samples/realistic-expected-output.json`：机器可读样例。
- `references/fields.md`：字段定义。
- `references/risk-rules.md`：风险规则。

## README 开头必须讲清楚

第一屏就要说明：

```text
This is a Codex Skill for cleaning Chinese WeChat group recruitment messages into JSON, tables, publishable copy, and risk warnings.
```

同时保留中文说明。

## 不要只发压缩包

压缩包适合发给熟人试用。

但要让 AI 智能体搜索和使用，必须发布到公开代码平台。

## 发布后要做

- 提交到 GitHub。

- README 加中英文关键词。
- 发布一篇中文介绍。
- 发给招聘群运营者试用。
- 收集真实微信群样本继续迭代。
